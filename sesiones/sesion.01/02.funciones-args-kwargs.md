### ðŸ”¹ `*args`: argumentos posicionales variables

Permite pasar cualquier nÃºmero de valores posicionales.
Internamente se reciben como una **tupla**.

```python

    return sum(args)

```

---

### ðŸ”¹ `**kwargs`: argumentos con nombre variables

Permite pasar pares `clave=valor` arbitrarios.
Internamente se reciben como un **diccionario**.

```python
def mostrar_info(**kwargs):
    for clave, valor in kwargs.items():
        print(f"{clave}: {valor}")

mostrar_info(nombre="Ana", edad=30, ciudad="Madrid")
# nombre: Ana
# edad: 30
# ciudad: Madrid
```

---

### ðŸ”¹ Combinando `*args` y `**kwargs`

Una funciÃ³n puede recibir ambos, primero `*args`, luego `**kwargs`.

```python

    print("args:", args)
    print("kwargs:", kwargs)

# args: (1, 2, 3)
# kwargs: {'usuario': 'david', 'rol': 'admin'}
```

### ðŸ”¹ Orden de los parÃ¡metros en una funciÃ³n

El orden correcto es:

1. Argumentos posicionales normales
2. `*args`
3. Argumentos con nombre por defecto
4. `**kwargs`

```python

    print(a, b, args, c, kwargs)
```
